메서드
	각각의 메서드는 혼용 사용이 가능
	
	
	




length() 문자열의 길이 - return int
charAt() 인덱스 위치에서의 문자 - return char
indexOf() 문자열을 검색해서 일치하는 인덱스 값 - return int
lastIndexOf() 문자열을 뒤에서부터 검색해서 일치하는 인덱스 값 - return int
String.valueOf(값) → 다른 타입을 String으로 변환 - return String

toUpperCase() 영문 문자를 모두 대문자로 변환 - return String
toLowerCase() 영문 문자를 모두 소문자로 변환 - return String
replace(a, b) a를 b로 치환 - return String
substring(a, b) 인덱스 a번부터 b-1번까지의 문자열 생성 - return String
split(기준자) 기준자를 기준으로 문자열을 분할한 배열 생성 - return String[]
trim() 문자열의 앞뒤 공백 제거 - return String

equals() 문자열의 실제 내용 비교 - return boolean (true, false)
equalsIgnoreCase() 대소문자 구분 없이 문자열의 실제 내용 비교 - return boolean

Integer.parseInt() 문자열을 int로 변환 - return int

클래스

클래스 멤버 - 필드, 생성자, 메서드



클래스(설계도, 물건을 만들기 위함)
필드(클래스 안의 변수, 전역변수)

객체 - 설계도를 사용해 만들어낸 완성품 (실제 사용)

필드 - 객체의 상태 (데이터)
생성자 - 객체를 처음 만들 때 실행
메서드 - 객체의 동작 (기능)



데이터 클래스: 설계도 (= 클래스)   ==> 객체(물건)
실행 클래스: 직접 객체를 생성하고 활용 (메인 메소드가 적어도 하나 이상 포함)




* 메소드

접근제한자 반환타입 메소드이름() {
	실행문;
	실행문;
	실행문;
	...
}


접근제한자: public
반환타입: void(없음) int double ...




반환(리턴) 타입:
*리턴은 무조건 하나를 반환한다


1. 반환값 없음
public void display() {
}


2. 반환값 있음
public int display() {
	return 0;
}

public double display() {
	return 0.0;
}

public String display() {
	return "";
}


3. 메소드 안으로 들어가는 값이 없는 경우, 매개변수가 없는 메소드
public void display() {
}


4. 메소드 안으로 들어가는 값이 있는 경우
   (매개변수를 이용해 메소드 안으로 값을 넣어줌)
public void display(int a) {  // 소괄호() 안의 매개변수를 사용
}

public void display(int a, int b) {
}

public void display(int a, double b) {   // 매개변수의 선언 순서는 상관 없지만, 호출할 때에 선언된 순서에 맞게 호출해야 함
}

public void display(double a, double b) {
}




전역 변수 = 필드. (클래스에서 선언한 변수)
지역 변수 = 메소드 안에서 선언한 변수





같은 클래스(설계도)를 사용한 경우, 
아파트처럼 구조 자체는 같으나 들일 수 있는 가구(변수 등)에 변화를 줄 수 있다





매개변수의 순서

선언 순서 자체에는 문법적으로 정해진 순서는 없음
가독성과 의미를 위한 '권장되는 순서'는 있음
메서드를 호출할 때는 선언한 순서를 반드시 그대로 따라야 함
	(double, int)로 지정했다면 호출할 때 double 타입, int 타입으로 순서를 지켜야 함


매개변수 = 인자
매개변수(parameter): 메서드 선언부에 쓰는 변수
인자(argument): 메서드 호출 시 실제로 전달하는 값

파라미터(parameter) =/= 아큐먼트(arqument)     // 정확한 의미 자체는 다름

그러나 매개변수, 인자, 파라미터, 아큐먼트라는 단어를 혼용하여 함께 사용함





	
	
* 생성자
1. 물건(객체) 생성 시 한 번 사용
2. 매개변수 X = 기본 생성자
   매개변수 O = 생성자 오버로딩과 관련(Overloading)
3. 생성자가 하나도 없으면, 기본 생성자는 자동으로 생성됨.
   (누가: 컴파일러가, 언제: 컴파일(저장)할 때)
4. 메소드와 비슷하지만, 차이점은 반환형이 없다. 클래스 이름과 동일하다.
5. this.필드명
   this: 현재 클래스 멤버(필드, 메소드)를 가리킨다.

public class A {
	public A() {
	}
	
	public void display() {
	}
}




2. 생성자의 종류
(1) 매개변수가 없는 생성자 - 기본 생성자   ()
	매개변수가 없는 생성자.
	클래스에 생성자를 하나도 정의하지 않으면, 자바 컴파일러가 자동으로 빈 생성자를 추가해 준다.

(2) 매개변수가 있는 생성자   (자료형)
	객체를 만듦과 동시에 원하는 값으로 필드를 초기화하고 싶을 때 정의한다.
